Template: When creating a Request for Comments (RFC):
1.  **Purpose:** Proposing major changes or new features requiring discussion.
2.  **Next Number:** Check `specs/rfc/` for correct numbering (start at 001 if empty).
3.  **Create File:** `specs/rfc/XXX-title.md`.
4.  **Structure:**
    *   **Problem:** Why do we need this?
    *   **Proposal:** Detailed design.
    *   **Alternatives:** What else did we consider?
5.  **Status:** Mark as `[Proposed]`.
6.  **Review:** Open PR and label `rfc`.
